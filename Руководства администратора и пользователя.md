# 1       Руководства администратора и пользователя

## 1.1 Руководство администратора

### 1.1.1  Введение

Руководство администратора предназначено для специалистов, осуществляющих установку, настройку, сопровождение и эксплуатацию системы Support System. Администратор отвечает за корректную работу серверной части, базы данных, Telegram-бота, а также за управление пользователями и контентом.

## 1.1.2  Требуемые ресурсы

Для корректной работы системы требуются следующие ресурсы:

**Аппаратные требования:**

·         процессор с не менее чем 2 вычислительными ядрами;

·         оперативная память – не менее 4 ГБ;

·         дисковое пространство – не менее 20 ГБ;

·         постоянное подключение к сети Интернет.

**Программные требования:**

·         операционная система Linux (рекомендуется Ubuntu 20.04 и выше);

·         Docker и Docker Compose;

·         Python версии 3.10 и выше;

·         СУБД PostgreSQL версии 14 и выше;

·         веб-браузер для доступа к административному интерфейсу.

### 1.1.3  Установка системы

Установка системы выполняется на сервере и включает следующие этапы:

1)      загрузка исходного кода проекта на сервер;

2)      настройка переменных окружения, включая параметры базы данных и токен Telegram-бота;

3)      запуск системы с использованием Docker Compose;

4)      выполнение миграций базы данных;

5)      создание учетной записи администратора.

После выполнения указанных действий система становится доступной для эксплуатации.

### 1.1.4  Настройка системы

Первичная настройка системы выполняется через административный веб-интерфейс Django Admin и включает:

·         настройку ролей и прав пользователей;

·         создание и редактирование категорий документации;

·         добавление оборудования;

·         загрузку документов и управление версиями файлов;

·         настройку параметров уведомлений администраторов.

### 1.1.5 Управление пользователями

Администратор имеет возможность:

·         создавать и удалять учетные записи пользователей;

·         назначать роли и права доступа;

·         управлять группами пользователей;

·         ограничивать доступ к административным функциям.

Все изменения фиксируются в журнале аудита системы.

### 1.1.6  Управление контентом

Администратор осуществляет:

·         создание и редактирование категорий документации;

·         управление иерархией категорий;

·         добавление карточек документов;

·         загрузку новых версий файлов документов;

·         контроль актуальности и доступности контента.

### 1.1.7 Резервное копирование и восстановление

Для обеспечения сохранности данных администратор обязан регулярно выполнять резервное копирование базы данных. Восстановление данных осуществляется путём загрузки резервной копии в СУБД и последующего перезапуска системы.

### 1.1.8  Аварийные ситуации

При возникновении аварийных ситуаций возможны следующие действия:

·         анализ журналов работы сервисов;

·         перезапуск остановленных компонентов системы;

·         проверка доступности базы данных и сетевого соединения;

·         получение уведомлений от системы мониторинга о сбоях в работе Telegram-бота.

## 1.2 Руководство пользователя

### 1.2.1 Назначение руководства

Руководство пользователя предназначено для сотрудников, использующих систему Support System через Telegram-бот для получения технической документации, инструкций и файлов.

### 1.2.2  Начало работы

Для начала работы пользователь должен:

·         открыть приложение Telegram;

·         найти Telegram-бот системы;

·         выполнить команду начала работы;

·         при необходимости пройти процедуру регистрации.

После этого пользователь получает доступ к функционалу системы в соответствии с назначенными правами.

### 1.2.3  Порядок работы пользователя

В процессе работы пользователь может выполнять следующие действия:

·         переходить по категориям документации;

·         просматривать список доступных документов;

·         выполнять поиск документов по ключевым словам;

·         получать актуальные версии файлов;

·         подписываться на категории для получения уведомлений об обновлениях.

Все действия выполняются с использованием кнопок и текстовых команд Telegram-бота.

### 1.2.4  Сообщения системы и реакция пользователя

В процессе работы система может выводить следующие сообщения:

·         сообщение об отсутствии найденных документов;

·         сообщение об ограничении прав доступа;

·         сообщение о временной недоступности данных.

В указанных случаях пользователю рекомендуется изменить поисковый запрос, повторить попытку позже либо обратиться к администратору системы.

### 1.2.5  Рекомендации по использованию

Для повышения эффективности работы пользователю рекомендуется:

·         использовать точные и краткие поисковые запросы;

·         сохранять полученные документы для офлайн-доступа;

·         использовать подписки для своевременного получения обновлений;

·         следить за уведомлениями системы.
