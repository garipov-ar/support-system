### 4.1.1 Методика проверки методического обеспечения
**Основание**: требование п. 1.4.3.8.
**Цель испытаний**: подтверждение наличия и полноты эксплуатационной документации.
**Порядок проведения**:
1. Проверяется наличие руководства пользователя и администратора.
2. Анализируется структура документов.
3. Проверяется наличие описания установки, эксплуатации и действий при сбоях.
4. Сопоставляется функционал документации с реализованным функционалом системы.
**Критерий соответствия**: Документация содержит все обязательные разделы и соответствует требованиям п. 1.3.8.

### 4.1.2 Методика проверки времени отклика

    def test_password_hashing(self):
        """Verify that passwords are not stored in plain text (п. 1.4.4.4.3)"""
        user = User.objects.get(username="specialist")
        # Django stores password as PBKDF2 hash by default
        assert user.password.startswith('pbkdf2_sha256$')
        assert user.check_password("specialistpassword123")
        assert not user.check_password("wrongpassword")

    def test_role_assignment(self):
        """Verify role assignment and group membership (п. 1.4.2.1.3)"""
        assert self.admin.is_superuser
        assert self.manager.groups.filter(name="Контент-менеджер").exists()
        assert self.specialist.groups.count() == 0
