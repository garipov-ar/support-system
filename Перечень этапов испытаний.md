- - 1.  Методика проверки методического обеспечения

**Основание**: требование п. 1.4.3.8.

**Цель испытаний**: подтверждение наличия и полноты эксплуатационной документации.

**Условия проведения**:

Проверка осуществляется комиссией в составе руководителя проекта и ответственного за разработку. Проверка проводится на основании утверждённой версии документации.

**Средства проведения**:

Текстовые документы системы, пояснительная записка, руководство пользователя, руководство администратора.

**Порядок проведения**:

1.  проверяется наличие руководства пользователя и администратора;
2.  анализируется структура документов;
3.  проверяется наличие описания установки, эксплуатации и действий при сбоях;
4.  сопоставляется функционал документации с реализованным функционалом системы.

**Обработка результатов**:

Результаты фиксируются в протоколе проверки с указанием соответствия/несоответствия.

**Критерий соответствия**:

Документация содержит все обязательные разделы и соответствует требованиям п. 1.4.3.8.

**4.1.2 Методика проверки времени отклика**

**Основание**: п. 1.4.2.2.1.

**Цель**: подтверждение соблюдения временного регламента.

**Условия проведения**:

Испытания проводятся при нагрузке до 100 одновременных пользователей в условиях стабильного интернет-соединения.

**Средства проведения**:

Средства логирования сервера, инструменты измерения времени отклика, тестовые аккаунты.

**Порядок проведения**:

1.  выполняется поиск информации через Telegram-бота;
2.  выполняется открытие категории;
3.  производится загрузка документа;
4.  фиксируется время ответа системы.

**Обработка результатов**:

Рассчитывается среднее и максимальное время отклика.

**Критерий соответствия**:

Время отклика менее 2 секунд.

**4.1.3 Методика проверки надежности и целостности данных**

**Основание**: п. 1.4.4.3.

**Цель**: проверка сохранности данных при штатной и аварийной работе.

**Условия проведения**:

Испытания проводятся на рабочем сервере с включённой СУБД PostgreSQL.

**Средства проведения**:

Средства управления БД, системные журналы, тестовые данные.

**Порядок проведения**:

1.  выполняется операция добавления документа;
2.  производится принудительное завершение работы сервера во время записи;
3.  выполняется восстановление системы;
4.  проверяется целостность данных и отсутствие дублирования.

**Обработка результатов**:

Анализируется состояние базы данных и журналы транзакций.

**Критерий соответствия**:

Данные не повреждены, дублирование отсутствует.

**4.1.4 Методика проверки требований безопасности**

**Основание**: п. 1.4.4.4.

**Цель**: подтверждение соответствия требованиям безопасности.

**Условия проведения**:

Испытания проводятся при подключении через защищённое соединение.

**Средства проведения**:

Инструменты проверки SSL-сертификата, средства просмотра кода, тестовые учетные записи разных ролей.

**Порядок проведения**:

1.  проверяется использование HTTPS;
2.  проверяется хранение паролей в хэшированном виде;
3.  проверяется разграничение прав доступа;
4.  проверяется журналирование действий.

**Обработка результатов**:

Фиксируются результаты проверки каждого пункта.

**Критерий соответствия**:

Все механизмы защиты функционируют корректно.

**4.1.5 Методика проверки адаптивности интерфейса**

**Основание**: п. 1.4.4.5.1.

**Цель**: проверка корректного отображения интерфейса.

**Условия проведения**:

Испытания проводятся на устройствах с различными разрешениями экрана.

**Средства проведения**:

Браузер с инструментами разработчика.

**Порядок проведения**:

1.  проверка при разрешении 360 px;
2.  проверка при разрешении 768 px;
3.  проверка при разрешении 1920 px.

**Обработка результатов**:

Оценивается корректность отображения элементов интерфейса.

**Критерий соответствия**:

Интерфейс корректно отображается при ширине более 360 px.

**4.1.6 Методика проверки резервного копирования**

**Основание**: п. 1.4.3.2.3.

**Цель**: подтверждение корректности создания и восстановления резервных копий.

**Условия проведения**:

Испытания проводятся в штатном режиме работы системы.

**Средства проведения**:

Средства резервного копирования, журналы выполнения задач.

**Порядок проведения**:

1.  проверяется создание резервной копии;
2.  выполняется удаление тестовых данных;
3.  производится восстановление из копии;
4.  проверяется срок хранения архивов.

**Обработка результатов**:

Анализируется успешность восстановления.

**Критерий соответствия**:

Восстановление проходит успешно, регламент соблюдён.

**4.1.7 Методика проверки защиты Системы от внешних воздействий**

**Основание**: п. 1.4.4.9.

**Цель**: проверка устойчивости системы к сбоям электропитания.

**Условия проведения**:

Испытания проводятся при наличии ИБП.

**Средства проведения**:

ИБП, средства мониторинга состояния сервера.

**Порядок проведения**:

1.  имитируется кратковременное отключение питания;
2.  проверяется автоматический перезапуск сервисов;
3.  измеряется время восстановления.

**Обработка результатов**:

Фиксируется время восстановления системы.

**Критерий соответствия**:

Восстановление ≤ 1 минуты.
